Platform Architecture

How NEPA works

NEPA separates perception into two non-blocking lanes for predictable, auditable operation.

01

Hard Deterministic Lane

Encode → Detect → Control. Fixed-cost operations, no heap allocation in hot path, preallocated buffers, lock-free queues, and non-blocking inter-lane communication.

02

Async Cognitive Lane

Semantic inference, classification, and context enrichment run asynchronously. This lane never blocks control output — inference latency cannot impact the deterministic path.

03

Structured Output

Detection events, risk classifications, and audit logs are emitted as structured data. Results are bit-for-bit identical across runs — reproducible and traceable.

04

Infrastructure Integration

Connects to operator toolchains via structured APIs. Compatible with MBIS-aligned workflows, asset management platforms, and inspection reporting systems.

About

AuraSense builds NEPA

AuraSense develops a neuromorphic-inspired inspection layer that bridges edge sensors and infrastructure analytics.

Neuromorphic-Inspired Inspection Layer

NEPA is a software inspection layer that integrates with autonomous drone systems and robotics platforms. It bridges edge sensors and infrastructure analytics through deterministic, bounded-latency processing.

Dual-Lane Architecture

The hard deterministic lane handles encode, detect, and control with fixed-cost operations. The async cognitive lane handles inference and enrichment — and never blocks control output.

Edge-First Deployment

NEPA is designed for edge deployment: CPU-only, offline-first, and deployable into existing inspection workflows without cloud dependency. It integrates with operator toolchains via structured SDK APIs.

Based in Hong Kong

AuraSense Limited is headquartered in Hong Kong, building infrastructure-grade middleware for the Greater Bay Area and global public-sector asset management markets.

Security & Data Handling

Compliance-oriented data handling

NEPA is designed for deployment in sensitive infrastructure environments with strict data governance requirements.

Offline-First Deployment

No data leaves the deployment environment. All processing, detection, and logging occurs on-device with no external network calls required.

Air-Gapped

Role-Based Access Control

Granular access control separates operator, inspector, and administrator roles. Access to detection outputs, audit logs, and configuration is governed by role assignment.

RBAC

bcrypt Password Hashing

All credential storage uses bcrypt with a configurable cost factor. Plaintext credentials are never stored or transmitted.

bcrypt

Ed25519 Output Signing

Detection outputs are cryptographically signed using Ed25519. Recipients can verify integrity and provenance of all inspection results.

Ed25519

Local Audit Logs

Tamper-evident local audit logs record all detection events, access attempts, and configuration changes for compliance and forensic review.

Audit Trail

Machine-Bound Licensing

License validation is bound to hardware identifiers. Runtime will not operate on unauthorized hardware, preventing unauthorized redistribution.
