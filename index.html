Problem

Infrastructure Inspection Constraints

Edge control systems face fundamental tradeoffs.

Latency Limits

Control decisions must complete within frame intervals. At 30 Hz, each frame arrives every 33.3 ms. Cloud round-trip latency (50-200 ms) exceeds available control budget, making real-time crack-aware navigation impossible.

Bandwidth and Storage Cost

Full video transmission at 1080p 30fps consumes 5.2 Mbps. Continuous operation generates 681 GB/day. At $8/GB cellular rates, this translates to $116,000/year per 1,000 km of infrastructure inspected.

Battery and Compute Tradeoffs

GPU-based inference draws 15-30W. CPU-only processing reduces power draw to 2-4W. On battery-constrained platforms, this difference determines flight duration and inspection coverage per charge cycle.

NEPA Architecture

Two-Lane System

Real-time control and semantic enrichment operate independently.

Lane 1

Real-Time Control

Implementation: C++

Latency Profile:

  • 2.50 ms average
  • 2.61 ms P95
  • 2.81 ms P99

SLA Violations: 0%

Safety Contract: 6 ms

Control Rate: ~106 Hz

Deterministic spike-based crack detection runs within bounded time. Decision output (FORWARD / CAUTION / SLOW) is available before next frame arrives.

Lane 2

Semantic Lane

Components: YOLO inference, signature enrichment

Execution: Non-blocking, asynchronous

Interference: Does not affect real-time control

Optional semantic analysis runs in parallel. YOLO-based object detection and metadata tagging operate without impacting Lane 1 latency guarantees. Results are logged for post-flight analysis.

SFSVC Layer

Sparse Frame-delta Spike Video Codec

Neuromorphic compression for edge transmission.

Frame Delta Encoding

Each pixel is compared to its previous value. Only pixels that cross a threshold generate events. Static regions produce zero output.

ON/OFF Spikes

Brightness increases generate ON events. Decreases generate OFF events. Binary representation reduces data volume while preserving temporal edges.

Sparse Representation

Typical scenes produce 6-10% non-zero spikes. Infrastructure surfaces with minimal texture generate even sparser outputs.

  • 9.97× compression ratio
  • 89.97% bandwidth reduction

Modeled Savings

Continuous operation at 1080p 30fps baseline: 681 GB/day transmitted. SFSVC reduces this to 68 GB/day.

  • 681 GB/day baseline savings
  • ~$116,000/year per 1,000 km inspected

Implementation Note

Public repository contains Python demo implementation (~8 ms/frame processing time). Production C++ SDK operates at sub-millisecond latencies and is not publicly released.

Control Output

Per-Frame Decision System

Each frame produces actionable control signals.

Crack Width

Estimated width in pixels based on spike band geometry. Mapped to physical dimensions using camera calibration parameters.

Severity Classification

Thresholds applied to crack width and spike density. Three levels: minor (monitoring), moderate (repair), severe (immediate action).

Confidence Score

Derived from spike consistency across adjacent frames. Low-confidence detections are flagged for human review.

Navigation Decision

Three states per frame:

  • FORWARD: No cracks detected, maintain speed
  • CAUTION: Minor crack, reduce speed for higher-resolution capture
  • SLOW: Severe crack, stop and perform detailed scan
