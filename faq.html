Licensing

Each license key is an Ed25519 cryptographically signed token issued by our licensing server. It is bound to your drone's hardware fingerprint on first activation. The SFSVC SDK validates the key locally — no internet connection required during flight (BVLOS compatible).

Each license tier supports a specific number of drone activations: Starter (1), Growth (4), Group (6), Enterprise (15). If you need more, upgrade your plan or contact us for a custom arrangement.

SFSVC will output a license expired message and refuse to initialise the pipeline until the subscription is renewed. Renewal is automatic if your subscription is active.

Technical

Any Linux x86_64 or ARM64 system with: 2+ CPU cores, 2 GB RAM, Ubuntu 20.04 or newer. Recommended: Intel N100 mini PC for optimal sub-2ms performance at 4.2W power draw.

No. SFSVC is designed specifically to run without a GPU. It uses SIMD CPU instructions (AVX2 on x86, NEON on ARM) for all processing. This is what enables sub-2ms latency at 4.2W — impossible on a GPU pipeline.

OpenCV is a general computer vision library that processes every pixel in every frame. SFSVC uses neuromorphic spike encoding to process only temporal changes — reducing data volume by ~98% before any detection occurs. This is why latency is under 2ms with no GPU, while OpenCV on equivalent hardware takes ~35ms.

ROS2 integration is on the roadmap for Q3 2026. Currently SFSVC provides a standard C++ API and can be called from any ROS2 node as a library. Contact us if you need early access to the ROS2 wrapper.
